Simon algorithm

Mar 13, 20252 min read

Consider the function:

fb:{0,1}n↦{0,1}nf_b:\{0,1\}^n\mapsto\{0,1\}^nfb​:{0,1}n↦{0,1}n

where bbb is not the null string, such that it presents a kind of periodicity:

fb(x)=fb(y)⇔x=y⊕bf_b(x) =f_b(y)\Leftrightarrow x=y\oplus bfb​(x)=fb​(y)⇔x=y⊕b

Meaning that the inputs can be grouped into couples that share the same output.

Algorithm

If we find two inputs belonging to the same output class, it’s easy to find b, as we can just XOR sum them.
Classically, we need to do a random search on the inputs, which has complexity exp⁡(n)\exp(n)exp(n).
In the quantum case:

  1. We have two registers A and B, containing nnn qubits each, initialized in ∣0⟩A⊗n∣0⟩B⊗n\ket{0}^{\otimes n}_A\ket{0}^{\otimes n}_B∣0⟩A⊗n​∣0⟩B⊗n​
  2. Apply a global Hadamard gate on A
  3. Apply a Uf gate on A and B
  4. Apply a global Hadamard gate on A
  5. Measure everything
    After the UfU_fUf​ gate, the state reads:
∑x∣x⟩A∣fb(x)⟩BN\sum_x\frac{\ket{x}_A\ket{f_b(x)}_B}{\sqrt{N}}x∑​N​∣x⟩A​∣fb​(x)⟩B​​

The last Hadamard on A and the measurement on B commute.
We perform the measurement first, so if we label the output classes in B fjf_jfj​, for each fjf_jfj​ we would have two states in A corresponding to the input associated to fjf_jfj​, i.e. ∣xfj⟩A,∣xfj⊕b⟩A\ket{x_{f_j}}_A,\ket{x_{f_j}\oplus b}_A∣xfj​​⟩A​,∣xfj​​⊕b⟩A​. Before the measurement:

1N/2∑j(∣xfj⟩A+∣xfj⊕b⟩A2∣fj⟩B)\frac{1}{\sqrt{N/2}}\sum _{j}\left(\frac{\ket{x_{f_j}}_A+\ket{x_{f_j}\oplus b}_A}{\sqrt{2}}\ket{f_j}_B\right)N/2​1​j∑​(2​∣xfj​​⟩A​+∣xfj​​⊕b⟩A​​∣fj​⟩B​)

So when we perform a projective measurement onto ∣fj⟩B\ket{f_j}_B∣fj​⟩B​, each output is equiprobable with probability 2/N2/N2/N.
After the measurement, on registry A:

∣xfj⟩+∣xfj⊕b⟩2\frac{\ket{x_{f_j}}+\ket{x_{f_j}\oplus b}}{\sqrt{2}}2​∣xfj​​⟩+∣xfj​​⊕b⟩​

We apply the Hadamard:

\frac{\ket{x_{f_j}}+\ket{x_{f_j}\oplus b}}{\sqrt{2}}\mapsto \frac{1}{\sqrt{2}}\frac{C}{\sqrt{N}}\sum _y (-1)^{x_{f_j}\cdot y}\left(1+(-1)^{b\cdot y}\right)\ket{y} $$where $C$ is a normalization constant (possibly $1$ but not sure and it's not important to the functioning of the algorithm). The only remaining states $y$ in the output of A are those orthogonal to $b$:

y\cdot b =0

Which k is enough to find $b$? It scales $\sim O(n)$.
